# Backlog do Produto

Esta seção apresenta o backlog do produto derivado dos requisitos funcionais, organizado em Histórias de Usuário e priorizado para definir o MVP (Produto Mínimo Viável).

## Backlog Geral

- **US01 - Cadastro e Acesso:** Como usuário novo, quero criar uma conta com meu e-mail e senha para que meus dados financeiros fiquem salvos e acessíveis apenas por mim.
- **US02 - Input Manual Rápido:** Como usuário, quero digitar rapidamente os dados de uma despesa ou receita no aplicativo, para que eu possa registrar gastos em dinheiro vivo ou transações que não geraram comprovante digital.
- **US03 - Importação Inteligente:** Como usuário, quero fazer upload de uma foto de nota fiscal ou arquivo, para que o sistema extraia os dados sozinho e eu não precise digitar tudo.
- **US04 - Correção de Lançamentos:** Como usuário, quero corrigir o valor de uma despesa que digitei errado, para que meu saldo fique correto.
- **US05 - Visão Geral (Home):** Como usuário, ao abrir o app, quero ver meu saldo atual e quanto já gastei neste mês, para ter uma noção rápida da minha saúde financeira.
- **US06 - Automação de Contas:** Como usuário, quero cadastrar contas fixas (ex: aluguel) para não precisar digitar isso todo mês.
- **US07 - Personalização de Categorias:** Como usuário, quero criar categorias com cores específicas, para diferenciar gastos visualmente nos relatórios.
- **US08 - Consulta Detalhada:** Como usuário, quero filtrar minhas transações para ver apenas o que gastei em uma categoria específica no mês passado.
- **US09 - Análise Visual:** Como usuário, quero ver um gráfico de pizza que mostre a distribuição dos meus gastos, para identificar onde estou gastando mais.
- **US10 - Controle de Metas:** Como usuário, quero definir um limite de gastos para uma categoria e ver uma barra de progresso para não estourar meu orçamento.
- **US11 - Gestão de Cartão:** Como usuário, quero registrar compras no crédito sabendo que o dinheiro só sairá da conta no dia do vencimento.
